# Auto-Generate Knowledge Docs — Design Document

## Goal

Let users auto-generate knowledge documents for their projects by spawning Claude to explore a codebase and produce a structured markdown summary. Agents use these docs as context for all tasks.

## Flow

1. User clicks "Auto-Generate" in Knowledge Docs tab
2. Dialog shows: dropdown of project paths from existing tasks + folder picker for any path
3. User selects a project path, clicks Generate
4. App spawns Claude in read-only mode pointed at the project directory
5. Claude explores the codebase and outputs a structured knowledge document
6. Output is saved as an `AIKnowledgeDoc` with `autoGenerated: true` and `sourcePath` set
7. If a doc already exists for that `sourcePath`, user is asked to confirm replacement

## Data Model

Add to `AIKnowledgeDoc`:
```typescript
sourcePath?: string  // Project path this doc was generated from
```

No other type changes.

## Backend

### New Module: `src/electron/ai-automation/knowledge-generator.ts`

- `generateKnowledgeDoc(projectPath: string): Promise<string>`
- Spawns Claude with `--print` mode, read-only tools (`Read,Glob,Grep,Bash(find:*),Bash(ls:*),Bash(cat:*),Bash(git:*)`)
- System prompt instructs Claude to explore and produce markdown covering:
  - Project purpose and overview
  - Tech stack and key dependencies
  - Architecture and file structure
  - Key patterns and conventions
  - Important configuration
- Captures and returns the full output text
- Same Claude spawning pattern as agent-runner but simpler (no task queue, no streaming)

### New IPC Handler: `aiGenerateKnowledgeDoc`

- Args: `[string]` (project path)
- Returns: `AIKnowledgeDoc`
- Calls generator, creates/replaces the knowledge doc in settings
- Matches existing docs by `sourcePath` field

## UI Changes

### Knowledge Docs Tab

- "Auto-Generate" button next to "Add Doc"
- Opens dialog with:
  - Dropdown of deduplicated project paths from existing tasks
  - Folder picker button for other paths
  - Generate button
- Loading state while Claude is working
- Confirm dialog if replacing existing doc for same project
- New/updated doc appears in list on completion
