# P9: Project Knowledge System — Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Give the planner structured knowledge about projects via a two-tier system — profiles in the system prompt, detailed docs on demand via MCP.

**Architecture:** Project profiles (Tier 1) stored as JSON in user data, injected dynamically into planner system prompt. Detailed knowledge (Tier 2) stored as markdown, served via `get_project_knowledge` MCP tool. Both generated by an agent scanning the project.

**Tech Stack:** Electron (fs, child_process), MCP tool handler, Claude CLI for generation

**Design doc:** `docs/plans/2026-03-24-project-knowledge-system-design.md`

---

### Task 1: Define types and storage

**Files:**
- Create: `types/project-knowledge.d.ts`
- Modify: `types/index.d.ts`
- Create: `src/electron/ai-automation/project-knowledge-manager.ts`

**Types:**

```typescript
interface ProjectProfile {
  projectPath: string
  name: string
  summary: string
  stack: string
  responsibilities: string
  generatedAt: string
}

interface ProjectKnowledge {
  projectPath: string
  architecture: string
  keyFiles: string
  apiIntegrations: string
  dataFlow: string
  development: string
  generatedAt: string
}
```

**Storage manager** — reads/writes to `{userData}/project-knowledge/{projectId}/`:
- `profile.json` — Tier 1 profile
- `knowledge.md` — Tier 2 full doc

Functions:
- `getProjectProfile(projectPath: string): ProjectProfile | null`
- `getAllProjectProfiles(): ProjectProfile[]`
- `saveProjectProfile(profile: ProjectProfile): void`
- `getProjectKnowledge(projectPath: string): string | null` (returns markdown)
- `saveProjectKnowledge(projectPath: string, markdown: string): void`
- `deleteProjectKnowledge(projectPath: string): void`

Use `Buffer.from(projectPath).toString('base64')` for the directory name (same ID scheme as DirectorySettings).

### Task 2: Build the generation agent

**Files:**
- Create: `src/electron/ai-automation/project-knowledge-generator.ts`

This spawns a Claude CLI agent that scans a project and produces both tiers.

**Generation prompt** — two-part system prompt:

Part 1 (Profile): "Produce a JSON object with these exact fields: name, summary (1-2 sentences), stack (comma-separated), responsibilities (2-3 sentences about what this project owns and when to involve it)."

Part 2 (Knowledge): "Produce a markdown document with these exact sections: ## Architecture, ## Key Files, ## API / Integrations, ## Data Flow, ## Development."

The agent should:
1. Spawn Claude CLI with `--print` pointing at the project directory
2. Parse the output — extract JSON profile and markdown knowledge
3. Save both via the storage manager
4. Return success/failure

Functions:
- `generateProjectKnowledge(projectPath: string, mainWindow: BrowserWindow | null): Promise<{ profile: ProjectProfile; knowledge: string }>`

Emit progress events to the UI via IPC (so the user sees "Generating knowledge for web-phoenix-journey...").

### Task 4: Make planner system prompt dynamic

**Files:**
- Modify: `src/electron/ai-automation/planner-prompt.ts`
- Modify: `src/electron/ai-automation/planner-runner.ts`

Change `PLANNER_SYSTEM_PROMPT` from a static string to a function:

```typescript
export function buildPlannerSystemPrompt(): string
```

This function:
1. Starts with the static instructions (current content of PLANNER_SYSTEM_PROMPT)
2. Calls `getAllProjectProfiles()` from the storage manager
3. If profiles exist, appends a "## Known Projects" section with each profile formatted as:

```
### {name}
- Path: {path}
- Summary: {summary}
- Stack: {stack}
- Responsibilities: {responsibilities}
```

4. Returns the complete prompt string

Update `planner-runner.ts` line ~111 to call `buildPlannerSystemPrompt()` instead of using the static `PLANNER_SYSTEM_PROMPT`.

Also update Step 2 in the prompt:
- "Review the Known Projects section below to understand what projects exist."
- "If you need deeper knowledge about a specific project, use `get_project_knowledge` with the project path."
- Remove the instruction to call `list_projects` for basic discovery (profiles are already in the prompt). Keep `list_projects` as a fallback.
